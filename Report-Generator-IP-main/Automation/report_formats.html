Below are the available report types and their structure/formatting details.

Initial IP(Type 1)

  • Summary of findings (IP-based vulnerabilities)
  • Table displaying IP addresses and the number of vulnerabilities severity wise
  • Risk assessment analysis table (per IP)
  • Detailed vulnerability tables (per IP, per finding)
  • IPs having no vulnerabilities are displayed in a separate table
  • Proof of Concept (steps, screenshots)
  • Remediation recommendations
  • Charts: Vulnerability distribution by severity for the entire assessment
  • Charts: Vulnerability distribution by severity for each IP

Initial Report (IP - Type 2)

  • Summary of findings (Vulnerabilities found in the IP addresses)
  • Table displaying IP addresses and the number of vulnerabilities severity wise
  • Detailed vulnerability tables (per finding, IP addresses are grouped together)
  • Vulnerability across multiple IPs is displayed in a single table
  • IPs having no vulnerabilities are displayed in a separate table
  • Proof of Concept (steps, screenshots)
  • Remediation recommendations
  • Charts: Vulnerability distribution by severity for the entire assessment

Cert-IN Report (Web)

  • Format as per Cert-IN guidelines for web applications
  • Document control pages (1, 4, 5, 6, 7) with structured data
  • Includes auditing team, activities, tools, and distribution lists
  • Excel template-based input for maximum efficiency
